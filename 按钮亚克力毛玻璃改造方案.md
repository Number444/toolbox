# Button 样式改造方案：仿亚克力毛玻璃（Acrylic / Frosted Glass）

> 目标：将 Toolbox 的 Button 样式从"实底 + 径向高光（亮面玻璃）"改为"半透明 + 磨砂质感（亚克力毛玻璃）"。
> 涉及文件：`App.xaml`（样式）；`App.xaml.cs` 与样式无关，方案 A 不需要改动。

---

## 一、现状分析

### 现有 Button 样式（App.xaml 第 29–162 行）

全局 `Button` 样式为实底绿（`#76B580`）+ 4 层渐变模拟的**亮面玻璃高光**：

| 层 | 名称 | 作用 |
|---|---|---|
| Layer 1 | `GlassReflection` | 径向点光源（右上角强光 + 四周深绿阴影） |
| Layer 2 | `InnerHighlight` | 受光面内高光（右上边缘白色发光） |
| Layer 3 | `SpecularHighlight` | 镜面反光带（右上角超高光亮条） |
| Layer 4 | `InnerShadow` | 背光面内阴影（左下边缘黑色压暗） |

这是 **glossy（亮面玻璃）** 气质，与毛玻璃的 **matte（磨砂哑光）** 气质相反。改造时应**整层替换**这 4 层，而不是叠加，否则又亮又糊会显得脏。

其他按钮样式：

- `ModeBtnStyle`（第 612–648 行）：胶囊形模式切换按钮，半透明灰底，需同步改造。
- `WindowButtonStyle` / `CloseButtonStyle`：标题栏按钮，**建议保持不动**（Win11 惯例中标题栏按钮为扁平透明）。

### 技术前提：WPF 没有控件级背景模糊

Win11 亚克力的配方是：**半透明底 + 背景高斯模糊 + 噪点 + 顶部切边高光**。
但 WPF 的 `BlurEffect` 只能模糊控件**自身**，无法模糊"身后的内容"（没有 `backdrop-filter` 等价物）。
因此按钮级 Acrylic 有两条路线：

- **方案 A（推荐）**：半透明拟态。窗口背景为深色 `#1C1C1C` 且基本静态时，"模糊"差异肉眼几乎不可感知，纯 XAML 即可达到约 95% 的观感，零依赖、零性能风险。
- **方案 B（进阶）**：真模糊，见第四节。

## 二、方案 A：半透明拟态亚克力（推荐落地）

### 第 1 步：新增画刷资源（放入 `Application.Resources` 颜色定义区）

```xml
<!-- 亚克力画刷 -->
<SolidColorBrush x:Key="AcrylicBaseBrush"        Color="#26FFFFFF"/>  <!-- 白 15% -->
<SolidColorBrush x:Key="AcrylicBaseHoverBrush"   Color="#40FFFFFF"/>  <!-- 白 25% -->
<SolidColorBrush x:Key="AcrylicBasePressedBrush" Color="#14FFFFFF"/>  <!-- 白 8%  -->
<LinearGradientBrush x:Key="AcrylicEdgeBrush" StartPoint="0,0" EndPoint="0,1">
    <GradientStop Offset="0" Color="#66FFFFFF"/>   <!-- 顶部入射光 -->
    <GradientStop Offset="1" Color="#14FFFFFF"/>
</LinearGradientBrush>
<!-- 可选：强调色变体，保留品牌绿 -->
<SolidColorBrush x:Key="AcrylicAccentBrush" Color="#4076B580"/>
```

### 第 2 步：重写全局 Button 模板

Style 级的 Setter 改动（关键两点：**文字改亮**、**启用渐变描边**）：

```xml
<Setter Property="Foreground" Value="{StaticResource TextPrimaryBrush}"/>  <!-- 原为 #1A1A1A -->
<Setter Property="Background" Value="{StaticResource AcrylicBaseBrush}"/>  <!-- 原为 AccentBrush -->
<Setter Property="BorderThickness" Value="1"/>                             <!-- 原为 0 -->
```

模板内用 **3 层哑光层**替换原来的 4 层高光 `Grid`：

```xml
<Border x:Name="bd"
        Background="{TemplateBinding Background}"
        BorderBrush="{StaticResource AcrylicEdgeBrush}"
        BorderThickness="{TemplateBinding BorderThickness}"
        CornerRadius="6"
        SnapsToDevicePixels="True">
    <Grid>
        <!-- 层1：竖向散射渐变（光在磨砂层内衰减，上亮下暗） -->
        <Border CornerRadius="5" IsHitTestVisible="False">
            <Border.Background>
                <LinearGradientBrush StartPoint="0,0" EndPoint="0,1">
                    <GradientStop Offset="0.00" Color="#17FFFFFF"/>
                    <GradientStop Offset="0.45" Color="#05FFFFFF"/>
                    <GradientStop Offset="1.00" Color="#00FFFFFF"/>
                </LinearGradientBrush>
            </Border.Background>
        </Border>

        <!-- 层2：噪点（Acrylic 磨砂颗粒感的来源，可选但强烈建议） -->
        <Border CornerRadius="5" Opacity="0.05" IsHitTestVisible="False">
            <Border.Background>
                <ImageBrush ImageSource="Resources/noise.png"
                            TileMode="Tile"
                            Viewport="0,0,128,128"
                            ViewportUnits="Absolute"/>
            </Border.Background>
        </Border>

        <!-- 层3：顶部 1px 切边内高光（Acrylic 标志性的"切面反光"） -->
        <Border Height="1" VerticalAlignment="Top"
                Margin="1,1,1,0" CornerRadius="5,5,0,0"
                Background="#3DFFFFFF" IsHitTestVisible="False"/>

        <!-- 内容层（Padding 通过 Margin 实现） -->
        <ContentPresenter Margin="{TemplateBinding Padding}"
                          HorizontalAlignment="Center"
                          VerticalAlignment="Center"
                          TextBlock.Foreground="{TemplateBinding Foreground}"/>
    </Grid>
</Border>
```

### 第 3 步：交互触发器（替换现有三个 Trigger）

| 状态 | 动作 |
|---|---|
| `IsMouseOver` | `bd.Background` → `AcrylicBaseHoverBrush`（只提亮底色，不加外发光） |
| `IsPressed` | `bd.Background` → `AcrylicBasePressedBrush`；可选加 `ScaleTransform 0.97` 按压回弹 |
| `IsEnabled=False` | `bd.Opacity` → `0.4`，同时 `Foreground` → `TextSecondaryBrush` |

参考写法：

```xml
<ControlTemplate.Triggers>
    <Trigger Property="IsMouseOver" Value="True">
        <Setter TargetName="bd" Property="Background" Value="{StaticResource AcrylicBaseHoverBrush}"/>
    </Trigger>
    <Trigger Property="IsPressed" Value="True">
        <Setter TargetName="bd" Property="Background" Value="{StaticResource AcrylicBasePressedBrush}"/>
    </Trigger>
    <Trigger Property="IsEnabled" Value="False">
        <Setter TargetName="bd" Property="Opacity" Value="0.4"/>
        <Setter Property="Foreground" Value="{StaticResource TextSecondaryBrush}"/>
    </Trigger>
</ControlTemplate.Triggers>
```

### 第 4 步：噪点纹理图

生成一张 128×128 灰度随机噪点 PNG，放入项目 `Resources/` 目录，Build Action 设为 **Resource**。
一行 Python 即可生成：

```bash
python -c "from PIL import Image; import random; Image.frombytes('L',(128,128),bytes(random.randrange(256) for _ in range(128*128))).save('noise.png')"
```

### 第 5 步：其他按钮样式配套

- **`ModeBtnStyle`**：`Background` 换 `AcrylicBaseBrush`，加 `BorderBrush="{StaticResource AcrylicEdgeBrush}"`、`BorderThickness="1"`，顶光层 `CornerRadius` 改为 `13` 适配胶囊外形。
- **`WindowButtonStyle` / `CloseButtonStyle`**：保持扁平透明，不做亚克力。

| 状态 | 底色 | 文字 | 其他 |
|---|---|---|---|
| 默认 | `#26FFFFFF`（白 15%） | `#F0F0F0` | 渐变描边 + 顶部切边光 + 噪点 |

## 四、方案 B：真模糊（进阶，备选）

### B1 · VisualBrush + BlurEffect 快照（控件级）

按钮背景用 `VisualBrush Visual="{Binding ElementName=背景根元素}"` 拍下身后内容，再叠 `BlurEffect Radius=20`：

```xml
<Border.Effect><BlurEffect Radius="20"/></Border.Effect>
<Border.Background>
    <VisualBrush Visual="{Binding ElementName=BackgroundRoot}" Stretch="None"/>
</Border.Background>
```

- 适用：悬浮窗压在一张**固定**背景图上。
- 缺点：对齐需手工计算 Viewbox/Viewport；背景滚动或变化时不会实时更新（除非定时重拍）。

### B2 · 窗口级 DWM Acrylic（整个窗口真模糊，性能最好）

P/Invoke 调用 `SetWindowCompositionAttribute`，参数 `ACCENT_ENABLE_ACRYLICBLURBEHIND` + 带 alpha 的 GradientColor（如 `0x991C1C1C`）。
项目 `Helpers.Win32Helper` 已有 P/Invoke 设施（`FindWindowByTitle`、`ShowWindow` 等），新增一个 `EnableAcrylic` 方法即可，在窗口 `OnSourceInitialized` 时调用。

- 效果：模糊的是**桌面**，作用于**整个窗口客户区**，是最接近 Win11 原生、性能最好的路线。
- 配合：窗口变真 Acrylic 后，按钮只需保持方案 A 的半透明拟态，透出的底就是真模糊。
- 代价：整个窗口（含文字区域）都会透出桌面，需谨慎调 alpha；远程桌面 / 旧显卡下可能降级为纯色。

### B3 · 换 UI 库

Wpf.Ui（lepo.co）或 ModernWpf 自带 AcrylicBrush 与整套 Fluent 主题。
代价：主题体系整体迁移，工作量最大。

## 五、注意事项

1. **文字必须改亮**：亚克力是深墙上的浅色玻璃，`Foreground` 必须用 `TextPrimaryBrush`（#F0F0F0），否则对比度不达标。
2. **整层替换，不要叠加**：现有 4 层亮面高光与新哑光层是相反气质，混用会显脏。
3. **性能**：不要给每个按钮常驻 `DropShadowEffect`（位图效果，按钮多了开销明显）；悬停态只提亮底色即可。方案 A 的所有层均为渐变画刷，渲染开销可忽略。
4. **禁用态**：半透明按钮只降 Opacity 会在深色墙上几乎消失，必须同时把字色换成 `TextSecondaryBrush`。
5. **兼容性**：若窗口 `AllowsTransparency=True`，某些旧显卡/远程桌面下 `BlurEffect` 会被跳过；方案 A 不依赖 BlurEffect，不受影响。
6. `App.xaml.cs` 与样式无关（仅负责单实例互斥与配置加载），方案 A 无需改动。
